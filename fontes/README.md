## Regras de uso

1. Cite o ID no módulo responsável sempre que uma afirmação sensível depender da fonte.
2. Diferencie o documento existir de o fato estar atualizado ou aplicável ao projeto.
3. Registre novo documento com o próximo ID disponível; não renumere os existentes.
4. Se o arquivo mudar, crie nova versão e registre a relação entre as versões.
5. Não mova documentos para `referencias/` quando forem evidências diretas da oportunidade.
6. Documentos marcados como confidenciais ou restritos não devem ser redistribuídos sem autorização.

## Política de arquivos grandes

- Use Git LFS somente para arquivos **maiores que 50 MB**.
- A regra deve ser específica para o caminho do arquivo; o Git LFS não seleciona automaticamente por tamanho.
- Antes de adicionar um binário, confirme seu tamanho. Arquivos de até 50 MB permanecem no Git normal.
- Atualmente somente `referencias/outros_fornecedores/deli/sem_data_deli_company_profile.pptx` ultrapassa o limite.
